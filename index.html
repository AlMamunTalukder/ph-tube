The scope of var, let, and const

var

This is function scope (means accessible within the function) and global scope (Means accessible outside of any function). var is hoisted means during compilation its go to top of the function. This value is reassigned.

let

This is Blocked-scope. Means they are only work in a fixed area like loop, if else. Its value also reassigned.

const

This is Blocked-scope. Its value isn't reassigned. For this, it is preventing accidental reassiging value.

The use cases of null and undefined

null

It is like that a array or object is define but their value is none. In further, if need value can set or reset.

undefine

It means value not assign or missing object property.
